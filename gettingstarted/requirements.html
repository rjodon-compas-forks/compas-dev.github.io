Dependencies

The COMPAS framework has very few dependencies, and most of them are optional. If you are happy working in Rhino or Blender, and you are not interested in or don’t need any of the numerical stuff, then everything should work out of the box; provided you have Python installed, of course.

The following dependencies are optional.

  • Numpy: For all numerical calculations and algorithms.

  • Scipy: For all numerical calculations and algorithms.

  • Matplotlib: For two-dimensional visualisations.

  • PyOpenGL: For three-dimensional visualisations.

  • PySide: For some of the standalone tools.

  • NetworkX: For spring layouts of networks.

  • Planarity: For planarity testing.

  • Numba: For just-in-time compilation.

  • PyCuda: For parallel computation through Nvidia’s CUDA.

  • PyOpenCL: For parallel computation though OpenCL.

  • Imageio: For reading and writing of image data.

Most of these are pip installable or ship with scientific Python distributions, such as Anaconda or Enthought EPD.

On Windows, many installers for remaining and otherwise difficult-to-install packages can be found on Christof Gholke’s page Unofficial Windows Binaries for Python Extension Packages.

On mac, installing whatever doesn’t ship with a scientific distribution is relatively easy with a package manager like macports or homebrew.
